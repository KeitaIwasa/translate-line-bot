# インターフェース仕様書

本ドキュメントは、LINE 多言語翻訳ボットの主要コンポーネント（LINE API / CloudFront / API Gateway / Lambda / Neon / Gemini）の連携仕様を示す。

## 1. 全体フロー概要

1. LINE Messaging API から Webhook イベントが API Gateway HTTP API に `POST` される。
2. API Gateway がイベントを Lambda 関数（Python 3.12）に渡す。
3. Lambda は署名検証後、イベントを解析し、Neon から言語設定と文脈を取得する。
4. Lambda は Gemini 2.5 Flash へ翻訳要求を送信し、Structured Output を受信する。
5. Lambda は翻訳結果を LINE Messaging API の Reply API または Push API に送信する。
6. Lambda が処理を完了後、API Gateway へ `200 OK` を返却する。

## 2. 外部公開インターフェース（Webサイト経由）

`kotori-ai.com` では CloudFront 同一オリジン構成を採用し、フロントエンドは以下を利用する。

- `GET https://kotori-ai.com/api/checkout`
- `POST https://kotori-ai.com/api/contact`
- `GET https://kotori-ai.com/api/stats/total-users`

補足:
- `api_base` / `apiBase` クエリは廃止。
- フロントの API 呼び出しは相対パス固定（`/api/...`）。

## 3. LINE Messaging API → API Gateway

- 本番Webhook URL: `POST https://h2xf6dwz5e.execute-api.ap-northeast-1.amazonaws.com/prod/webhook`
- ステージングWebhook URL: `POST https://cbvko1l0ml.execute-api.ap-northeast-1.amazonaws.com/stg/webhook`
- 認証: `X-Line-Signature`
- ヘッダー:
  - `Content-Type: application/json`
  - `X-Line-Signature: <signature>`

## 4. CloudFront → API Gateway（同一オリジン）

- CloudFront は `/api/*` を API Gateway に転送する。
- viewer-request Function で `/api` プレフィックスを除去してから API Gateway に渡す。
- 例: `/api/contact` → `/contact`

## 5. API Gateway → Lambda

- 統合タイプ: Lambda プロキシ統合（HTTP API）
- 主なルート:
  - `POST /webhook`
  - `POST /stripe/webhook`
  - `GET /checkout`
  - `GET /stats/total-users`
  - `POST /contact`
  - `OPTIONS /contact`

## 6. Lambda → Neon (PostgreSQL)

- 接続方式: `psycopg`
- 接続文字列: `NEON_DATABASE_URL`
- 主利用テーブル:
  - `group_members`
  - `messages`
  - `group_subscriptions`
  - `contact_rate_limits`

## 7. Lambda → Gemini 2.5 Flash

- エンドポイント（Google API）
- 認証: `GEMINI_API_KEY`
- Structured Output(JSON)で翻訳結果を取得
- 失敗時は `TranslationRetry` 設定に従って再試行

## 8. Lambda → LINE Messaging API

- Reply: `POST https://api.line.me/v2/bot/message/reply`
- Push: `POST https://api.line.me/v2/bot/message/push`
- 認証: `Authorization: Bearer <LINE_CHANNEL_ACCESS_TOKEN>`

## 9. CORS / セキュリティヘッダ

- Contact API は `CONTACT_ALLOWED_ORIGINS` に基づいて `Access-Control-Allow-Origin` を返す。
- 本番では `https://kotori-ai.com` のみ許可。
- `Access-Control-Allow-Origin: *` は採用しない。
- 配信レイヤ（CloudFront）で以下を付与:
  - `Referrer-Policy: no-referrer`
  - `Content-Security-Policy`（`connect-src 'self'`）

## 10. 主要な環境変数・秘密情報

| キー | 用途 |
| ---- | ---- |
| `LINE_CHANNEL_SECRET` | Webhook 署名検証 |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE API 送信用 |
| `NEON_DATABASE_URL` | PostgreSQL 接続 |
| `GEMINI_API_KEY` | Gemini API 呼び出し |
| `SUBSCRIPTION_FRONTEND_BASE_URL` | Pro導線のフロントURLベース |
| `CHECKOUT_API_BASE_URL` | 旧互換用（同一オリジン運用では通常空） |
| `SUBSCRIPTION_TOKEN_SECRET` | 署名付きトークン生成 |
| `CHECKOUT_SESSION_SECRET` | Checkoutセッション暗号用途 |
| `CONTACT_IP_HASH_SALT` | Contactレート制限用ハッシュ |

以上。
